# Project 02: Transaction Monitoring and Suspicious Activity Analysis

## Project Overview

This fictional portfolio project demonstrates the review of customer transaction activity after onboarding.

The project involves analysing a retail business customer whose account activity begins to differ from the expected customer profile.

The assessment considers:

- Expected versus actual transaction activity
- Unusual transaction patterns
- Third-party payments
- Rapid movement of funds
- Round-number transactions
- Cross-border activity
- Customer risk indicators
- Whether escalation is required
- Appropriate post-review controls

## Scenario

A fictional retail business customer was previously onboarded as a moderate-risk customer.

The customer stated that the account would be used mainly to make payments to South African suppliers for household and consumer goods.

During ongoing monitoring, several transactions were identified that may not be consistent with the original customer profile.

The purpose of this project is to determine whether the activity is reasonable, requires clarification, or should be escalated for further investigation.

## Project Objective

The project will demonstrate how to:

1. Review transaction activity
2. Compare transactions with the expected customer profile
3. Identify potential AML red flags
4. Document investigative reasoning
5. Determine whether enhanced review is necessary
6. Prepare an internal escalation recommendation

## Planned Project Files

1. Customer Profile
2. Transaction Dataset
3. Transaction Analysis
4. Red-Flag Assessment
5. Internal Escalation Report
6. Final Case Conclusion
